# Post-audit tiebreaker

Research snapshot: 2026-07-19.

## Decision

**Hackathon bet: capability settlement.** Build it only as a falsifiable promotion experiment: an assisted task can be complete while the capability claim stays provisional until a delayed, independently scored near- or far-transfer event settles it.

**Runner-up: authority-continuity dependency receipt.** It is the best second experiment because its authority rule, adversarial cases, deterministic baselines, and external outcomes can all be assembled in a temporally sealed local benchmark. It is a runner-up hackathon mechanism, not an automatic fallback product if capability settlement fails.

**Raw ceiling leader: function-grounded manufacturing ambiguity witness.** If an authoritative critical-to-quality predicate, deterministic CAD/tolerance proof, and real engineer-adjudicated cases were already available, it would have the highest technical and visual hackathon ceiling. Those inputs are the unproven gate, not an implementation inconvenience. Public NIST assets can support a conformance demonstration but cannot establish that a seeded omission was a real design-intent defect or that the paired witness adds lift over ordinary tolerance analysis.

**Standalone-category verdict: none of the four is proven to be a defensible standalone category.** Capability settlement may collapse into an LMS mastery feature; the dependency receipt into a source allowlist plus provenance hook; manufacturing into a CAD/tolerance-analysis plugin; and private cohort pay reconciliation into a feature of a worker, union, or pay-audit platform. A strong hackathon entry can still test one of those residual mechanisms. Winning-ceiling evidence is not category evidence.

The broad ADE / causal-history thesis remains disqualified and is not reconsidered here.

## How the tie was broken

The four official criteria are scored equally from 0–4. The numbers below are **credible ceilings for an honest, complete implementation**, not scores for the current repository, which contains research rather than a runnable submission. Literal Stage Zero therefore fails for every candidate today. The useful selection question is which concept has a credible path to passing it.

Truth and authority are pass/fail gates before the totals. Validation access asks whether the team can obtain a non-model oracle, the necessary decision authority, affected-user evidence, and the strongest baseline comparison. It does not reward short build time.

The 13–13 tie does not make manufacturing the bet. If both concepts had already passed their gates and actually achieved these ceilings, manufacturing's 4 in Technological Implementation would give it the official listed-order tie-break over capability settlement's 3. Selection happens earlier, however: capability settlement has the stronger causal evidence, the lower authority burden, and materially better access to a valid outcome study. A ceiling that depends on unavailable truth and validation is not the best current bet.

## Binding substitute and fatal-flaw review

### Capability settlement

The problem evidence is the strongest of the four: in the cited preregistered field experiment, unrestricted assistance improved immediate work while later unassisted performance fell in the studied setting. The result does not automatically generalize beyond that setting, but it directly establishes the proxy failure the mechanism changes. The demo state is also complete and safe: `task complete / capability provisional` changes only after an independently scored transfer action, with `capability not measured` available when no valid probe exists.

The named substitutes bind the novelty score at 2. Khanmigo already measures next-item correctness without the tutor; LMS mastery states, delayed assessment, AI vivas, adaptive probes, and practical performance checks are familiar. The only defensible residue is a portable, assistance-tool-independent settlement object, delayed far transfer, privacy-preserving evidence boundaries, and probe generation that demonstrably lowers expert work or improves validity.

Its fatal flaw is **ordinary mastery-assessment collapse**. Kill it if a teacher-authored delayed item bank plus an ordinary LMS mastery state produces equivalent transfer, calibration, teacher effort, and learner decisions; if learners experience it as a redundant second grade; if valid far-transfer probes require prohibitive expert correction; or if accommodations make the settled state misleading. GPT-5.6 may decompose offloaded operations and adversarially diversify probes, but it may not certify capability or grade its own work.

### Authority-continuity dependency receipt

This candidate has the cleanest accessible adversarial benchmark. A plausible package name can be published after an agent first hallucinates it, carry valid provenance to an attacker repository, expose the expected shallow API, and visibly execute a harmless sandbox marker. The receipt can then demonstrate the exact changed state: provenance-only admission executes the wrong package, while a preexisting source-authority mismatch yields `IDENTITY UNRESOLVED` and leaves the protected lockfile unchanged.

The substitute set binds both the implementation and idea claims. Capability-based package resolution is old; PySelect and adjacent systems already map natural-language needs to packages; npm/SLSA/Sigstore bind source, build, publisher, and artifact; and Snyk, Socket, and Phylum already occupy agent or pre-install policy paths. The residual is not semantic package addressing, recommendation, provenance, health scoring, or enforcement. It is continuity from an authority edge established **before** candidate discovery through source, publisher, artifact, and bounded executed behavior.

Its fatal flaw is **authority-bootstrap / allowlist collapse**. If registry metadata, candidate documentation, provenance, or a candidate-controlled repository can establish authority, the attacker can manufacture the complete story. If an independent source root is mandatory, open-ended discovery cannot autonomously authorize and the product may be only an evidence-rich allowlist. Kill it if a human-authored source allowlist plus deterministic verification matches admission outcomes without materially more human work, if legitimate discovery becomes per-package manual approval, or if GPT-5.6 does not reduce contract and witness authoring at equal accuracy. This is why it is second despite excellent validation access and why it is not a proven category.

### Function-grounded manufacturing ambiguity witness

This has the most impressive honest implementation ceiling. GPT-5.6 can reconcile multi-sheet drawings, CAD/PMI, notes, and assembly context; a CAD/tolerance solver can construct two minimal source-conforming interpretations; independent tools can certify both variants and evaluate a predeclared CTQ; and an engineer can turn the witness into a signed release-state transition. No reviewed incumbent explicitly advertises that exact paired constructive witness.

The fatal flaw is the **materiality oracle**. Different conforming parts normally prove permitted variation, not a defect. Materiality must come from a separately authoritative CTQ. Once that CTQ is supplied, F4-, Autodesk-, CETOL-, Siemens-, Capvidia-, and related checking or tolerance systems already perform much of the causal work. AI drawing reviewers and recent annotation-to-feature research further narrow the residue. The paired artifact is distinct only if it discovers consequential underdetermination that the best checker-plus-tolerance baseline misses and lets engineers resolve it faster than a precise warning.

Access, rather than build time, removes it from the top two. NIST test cases do not supply a ready ambiguity benchmark and explicitly cannot turn every missing tolerance into a defect. Advance it only with responsible-engineer CTQs, deterministic proof, historical cases, and a blinded fixed-false-block comparison. Kill it if the CTQ states the missing requirement so directly that tolerance analysis finds the same issue, if the pair is ordinary tolerance variation, if engineers prefer the direct warning, or if GPT-5.6 ablation removes no meaningful true positives.

### Worker-governed private cohort pay reconciliation

This candidate has the strongest human stakes and could change bargaining evidence rather than merely produce advice. The narrow mechanism compares transformation patterns locally, suppresses small cells, reveals nothing until a safe threshold is met, and lets workers decide whether a cited packet goes to an advocate. It must never label a discrepancy unlawful or intentional.

The novelty gap is already compressed. WageWatch explicitly addresses coworker group claims, and the Shipt Calculator deployment aggregated worker pay data with nonprofit worker organizations. Individual timesheets and pay-audit apps occupy the reconstruction layer. The residue is privacy-preserving pattern comparison, thresholded release, and worker governance—not collective pay auditing in general.

Its fatal flaw combines **prior-art compression with retaliation and re-identification risk**. The network needed to create corroboration is also a sensitive target. Strong encryption, local extraction, secure aggregation or an audited equivalent, minimum-cell rules, deletion, informed consent, and a threat model are product truth, not optional polish. Without a paid worker-center or union partnership and consented affected-worker validation, the demo can prove only that sample rows match; it cannot prove safety, governance, actionability, or a changed correction outcome. Severe harm cannot compensate for that failed access gate.

## Why the runner-up is not the raw-ceiling leader

The dependency receipt ranks second because its decisive truth test is accessible and adversarial: precommitted authority, signed artifacts, sandbox execution, hidden compatibility assertions, and a source-allowlist baseline can all produce externally checkable outcomes. That is validation access, not an assumption that security software is easier to build.

Manufacturing would replace it as runner-up—and directly challenge the bet—only after obtaining real engineer authority, historical cases, and the incumbent deterministic baseline. Private cohort pay would replace it only after a worker organization accepts governance responsibility and confirms that the privacy-thresholded residual can be gathered and acted on safely. Neither access condition is evidenced in the reviewed record.

## Promotion tests

### Bet: capability settlement

Run a blinded bounded-domain comparison of:

1. ordinary LMS/mastery state plus a teacher-authored delayed item bank;
2. portable settlement with teacher-authored probes;
3. portable settlement with smaller-model-generated probes; and
4. portable settlement with GPT-5.6-generated near- and far-transfer probes.

Use independent deterministic or blinded-human scoring. Measure delayed near transfer, far transfer, retention, confidence calibration, learner decisions, learner burden, educator correction time, and accommodation validity. Promote the hackathon build only if the portable state changes a consequential learner or educator decision, or GPT-5.6 materially reduces expert authoring at equal validity, without surveillance or punitive use.

### Runner-up: authority-continuity receipt

Run the temporally sealed slopsquat benchmark against registry-existence, provenance-only, available current pre-install gates, and a human-authored source allowlist plus the same deterministic verifier. Include rare legitimate packages, multiple honest providers, no-new-dependency cases, and unanchored discovery. Promote only if the receipt prevents wrong-authority execution while retaining legitimate admissions and GPT-5.6 materially reduces human contract or witness work. Otherwise retain it, at most, as deterministic security infrastructure.

Failure of the capability-settlement promotion test does **not** automatically promote the dependency receipt, manufacturing witness, or worker cohort mechanism. Each has an independent fatal flaw and must pass its own gate.

## Final call

Choose **capability settlement** as the sole evidence-adjusted hackathon bet. Keep the **authority-continuity dependency receipt** as the runner-up experiment because it has the next-best access to an honest falsifier, not because it has the second-highest rubric ceiling. Record **function-grounded manufacturing ambiguity witness** as the highest raw hackathon ceiling and a validation-blocked wildcard. Do not advance **worker-governed private cohort pay reconciliation** without real worker-organization governance.

None is yet a proven standalone category. The selected bet earns one promotion experiment, not a category declaration.
