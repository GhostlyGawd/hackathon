# JRN-02 verification evidence

JRN-02 implements named teacher and student journey authoring at source commit `a952d3326417e682b666dbd21798e28a3dc79ad3`. A saved version contains the fictional actor, goal, start state, current human-confirmed rule, current authorization, bounded allowed and prohibited actions, fictional fields, required visible checkpoints, steps, and immutable lineage. The UI and API expose the causal path from the confirmed rule through each fictional field to its required checkpoint.

## Red-first record

The manifest records focused failures before the builder, service, PostgreSQL repository, migration, signed route, and browser behavior existed. Later regression reds proved that agreement history could leak across scope, review-due authority could be mislabeled current, the database could accept an out-of-scope action, old fixtures could bypass the named-journey contract, and a standalone request context could lose the production browser session. The evidence harness also rejected both curated screenshots until this manifest referenced them.

## Claim boundary

This task defines and versions journey specifications; it does not execute, replay, or repair them. Those behaviors belong to JRN-03 and RUN-04. `RUNNABLE` means only that current authoring prerequisites are complete. It does not mean a test ran, passed, found no issue, or established safety or compliance.

All evidence uses controlled fictional accounts and reserved `.invalid` destinations. No real student records, production accounts, credentials, or public accusations are present.
