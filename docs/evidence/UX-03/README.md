## What this task proves

- An operator sees the named fictional journey, role, allowed origin and actions, source-bound recorder frame, recorder facts, checkpoint coverage, and canary category while a run is active.
- The model action summary and deterministic recorder event occupy separate labeled lanes. No chain-of-thought is shown or used as evidence.
- An authorized operator can stop the active run by keyboard. The server finalizes a CANCELED manifest from recorder-owned facts, preserves the observed checkpoint, and marks the unfinished checkpoint NOT TESTED.
- Finding labels expose exact scope, visibility, limitations, deterministic evidence, and excluded model prose before the receipt.
- A receipt is decision-ready only when exact scope, independent validity, and a named next human action are present. All eight review questions stay in order.
- A witnessed-conflict receipt can move an existing approval to HOLD. Automation never restores approval; the signed keep-hold and restore paths remain human-owned and append-only.
- Completed, partial, failed, worker-integrity-failure, canceled, and frozen retry records keep distinct meanings and exact lineage.

## Claim boundary

The live preview is a source-bound recorder frame, not an interactive embedded browser. A valid receipt proves included bytes have not changed; a sampled run, even when clean, does not prove safety, compliance, approval, or behavior outside the named scope. All evidence uses only the controlled fictional fixture and reserved test destinations.
