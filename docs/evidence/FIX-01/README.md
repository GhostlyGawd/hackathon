# FIX-01 verification evidence — complete

FIX-01 is complete at implementation source commit `6c9657f3fadb1721a6816327b6d30eb94fafc4ae`. It provides a deterministic, fictional classroom fixture with nine independently scored modes for baseline, witnessed conflict, repair, ambiguity, visibility loss, interface drift, prompt injection, risky action, and service failure.

The fixture is mechanism evidence, not a real-vendor finding, legal conclusion, safety certification, compliance determination, adoption result, or market validation. Its oracle is unavailable through package exports and HTTP routes, but repository administrators can read the file; blinded evaluations must withhold it from the application evaluator.

## Behavior implemented

- A seed deterministically creates reserved `.invalid` teacher and student identities, a unique class phrase, a response canary, and the exact fictional DPA rule.
- Real browser submissions reach controlled reserved hosts through a Chromium resolver rule, and the fixture records the requests it actually receives rather than manufacturing an expected trace.
- The independent oracle defines the bounded expected result, destinations, exact email recipients, transforms, visibility, journey shape, and real-world dispatch for all nine versions. It is neither exported from the package nor served over HTTP.
- Regression mode sends the exact synthetic student email to fictional analytics; repaired mode sends only an aggregate completion event; ambiguous and invisible modes remain explicitly unknown.
- Page instructions cannot expand authority. The risky-action path stops before contacting a real person, failure mode dispatches nothing, non-seeded input is rejected, and all process data is ephemeral.

## Known limitations

- A deterministic fixture proves seeded mechanism behavior only. It says nothing about a real vendor, legal meaning, safety, compliance, adoption, or market impact.
- Repository administrators can read the independent oracle; blinded runs must exclude it from the evaluated application path.
- Reserved destination hosts require the documented controlled-Chromium resolver rule and are not public DNS names.
- The fixture records its own actual seeded requests. RUN-02 owns the independent recorder, DET-03 owns findings, and DET-05 owns approval holds.
- Fixture state intentionally resets when the process restarts.
